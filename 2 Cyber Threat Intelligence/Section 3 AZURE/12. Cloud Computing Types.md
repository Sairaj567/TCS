Then let's also define the terms public, private, multi and hybrid cloud.

And the first one probably the most intuitive one is the so-called public cloud.

The most prominent offerings in this case are originating from Microsoft with their Azure catalog.

Then we have Amazon with AWS and we have Google with GCP.

And all of those three can be considered public cloud environments, meaning all the properties that

we discussed earlier are fulfilled and those resources are shared with the general public.

Then we have the private cloud offerings.

This is, again, those three big providers offering, um, a majority of their cloud solutions, even

in a private environment, meaning a company can leverage Azure Stack in their data center.

They can also do this with AWS Outposts or Google Anthos.

And if we combine those solutions, we essentially operate in a so-called hybrid cloud, meaning if

you have, um, certain Azure services that you leverage in the public cloud offering of Microsoft,

but you also have Azure Stack in your data center, this would be considered a hybrid cloud and then

also a very common pattern across a lot of cloud customers is the fact that you do not only operate

maybe in Azure, um, plus leveraging Azure Stack, you may also have workloads, for example, in AWS.

And this is what we commonly refer to as a so-called multi-cloud.

The scenario with especially the bigger enterprise environments is that you have public cloud offerings.

In most cases in a multi-cloud environment, meaning an enterprise would leverage maybe all three.

So Azure, AWS and GCP, but then they would also operate some of those offerings in a private cloud,

for example, Azure Stack.

And if you combine all of those, again you have a hybrid cloud.

So essentially the bigger enterprise customers have a public a private as well as a hybrid cloud and

a multi-cloud because they simply leverage a lot of those offerings together.
